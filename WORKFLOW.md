# 每小時迭代工作流

目標：每一輪都能產出可上線的小改版，確認沒壞掉後立刻推到 GitHub Pages。

## 固定節奏

### 第 0-10 分鐘：確認目標

- 看線上站台是否正常。
- 檢查目前資料欄位與 API 狀態。
- 從 backlog 挑 1-2 個本輪一定能完成的項目。

### 第 10-35 分鐘：實作

- 小步修改介面與資料轉換層。
- 每個資料欄位都要標記來源；沒有真資料時顯示等待資料源，不塞假數字。
- 優先做手機版可讀性。

### 第 35-45 分鐘：驗證

- 本機開 `http://127.0.0.1:4173/`。
- 檢查首頁、賽程、排名、數據、收藏、單場詳情。
- 至少確認 console 沒有錯誤。

### 第 45-55 分鐘：發布

- `git status`
- `git add`
- `git commit -m "..."`
- `git push origin main`
- 確認 GitHub Pages 狀態為 built。

### 第 55-60 分鐘：紀錄

- 更新本文件的迭代紀錄。
- 寫下一輪要做什麼。
